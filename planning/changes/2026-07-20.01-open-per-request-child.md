# Change: Open the per-request child container (modern-di 3.x mandatory-open)

**Lane:** lightweight — one real adapter bug fix (1 line in `main.py`), plus
the test-suite adaptations it requires. Maintainer-ruled lane per the
modern-di 3.x wave.

## Goal

Bump `modern-di` to `>=3,<4`. Under 3.0's mandatory-open lifecycle a
freshly-constructed (or freshly-built-child) container starts unopened;
resolving from it, or building a child of it, before `open()` raises
`ContainerClosedError`. `_build_child` (`modern_di_grpc/main.py`) built the
per-request `REQUEST` child and returned it unopened — the four interceptor
wrappers only ever *close* it (`close_sync`/`close_async`), never open it.
Every RPC hit `ContainerClosedError` on its very first resolve.

## Approach

**Part A — adapter fix.** `_build_child` now calls `child.open()` right after
`build_child_container`, before returning it. `.open()` is sync and correct
on both the sync and aio paths (resolution itself is sync-only in modern-di;
only the *close* side varies, `close_sync`/`close_async`, which was already
correct). One change in `_build_child` covers all four wrapper builders
(`_wrap_unary_sync`, `_wrap_stream_sync`, `_wrap_unary_aio`,
`_wrap_stream_aio`), since they all route through it.

**Part B — root stays caller-owned.** Building a child requires the *root* to
be open too — `build_child_container` raises `ContainerClosedError` on a
closed parent. gRPC gives `DIInterceptor`/`DIAioInterceptor` no
root-lifecycle hook (no server start/stop callback), so — consistent with
the maintainer's `modern-di-flask` ruling — the **caller** opens the root,
not the interceptor's `__init__`. No adapter code changes for this part; it's
already the caller's job to close the root, and now it's also the caller's
job to open it first. See
[architecture/dependency-injection.md](../../architecture/dependency-injection.md#user-owned-root-lifecycle).

Every test that builds a root `Container` and feeds it to an interceptor (or
to `_build_child`/`build_child_container` directly) now opens it: `.open()`
where the fixture/test already manages an explicit teardown sequence that a
`with` block would disrupt (e.g. a root whose `AppResource` finalizer is
*async*, so it must never go through `close_sync()` — wrapping it in a plain
`with` would raise `AsyncFinalizerInSyncCloseError` at fixture teardown).

## Files

- `modern_di_grpc/main.py` — `_build_child` opens the child before returning.
- `tests/test_sync.py` — `sync_channel` fixture and
  `test_app_finalizer_runs_on_root_close` open the root explicitly.
- `tests/test_aio.py` — `_make_server` and
  `test_wrap_unary_aio_resets_context_var_when_close_raises` open the root.
- `tests/test_inject.py` — `_rpc_child`, `test_build_child_seeds_servicer_context`,
  `test_close_runs_app_and_request_finalizers`, and
  `test_wrap_unary_sync_resets_context_var_when_close_raises` open the root
  and/or child before use.
- `architecture/dependency-injection.md` — updated "Per-RPC scope" and
  "User-owned root lifecycle" for the mandatory-open lifecycle.
- `README.md` — usage example opens the root before constructing the
  interceptor; migration note added.
